## What P2 Added

- **Emphasis state via Mercutio.** A new cast member `Mercutio, a tracker of emphasis state.` mirrors Ophelia exactly: `0` = closed, `1` = open. Phase 3 adds `@TOGGLE_EM`, `@OPEN_EM`, `@CLOSE_EM` scenes that swap Juliet out for Mercutio to read and flip the flag, then swap Juliet back in to push the `<em>`/`</em>` chars onto Oberon. Dispatch in `@EMIT_LOOP` fires on `*` (42) or `_` (95). No changes to the inline IR — `<em>` chars are emitted character-by-character from the same emit pipeline as everything else.
- **Blockquote detection in Phase 2.** `@SEED` peeks Puck's top with a Recall-and-restore pattern: pop the first char, compare to `>` (62), and either restore it (normal paragraph path) or consume it and try to also consume one following space. On the blockquote path, Mercutio is stage-swapped in briefly and set to `1` to flag end-of-input emission; the raw chars of `<blockquote>\n` are pushed onto Hamlet ahead of the paragraph-open marker so Phase 3 emits them before `<p>`.
- **Blockquote close via raw chars.** `@FINISH` dispatches on Mercutio's flag: non-blockquote pushes the `2` paragraph-close marker as usual (emits `</p>\n\n`); blockquote pushes the raw chars of `</p>\n</blockquote>\n` directly onto Hamlet, bypassing the `2` marker to avoid the extra trailing blank that `@EMIT_P_CLOSE` emits. `@FINISH_END` resets Mercutio to `0` before `[Exeunt]` so Phase 3's emphasis state starts clean.
- **Backtracking is not implemented.** `p2_backtrack_input.md` is `One *foo **bar* baz** here.`; Markdown.pl outputs `One <em>foo <strong>bar</em> baz</strong> here.` with overlapping `<em>/<strong>` tags. A single 0/1 toggle cannot reproduce that. Test is marked `xfail` with a reason pointing to this document.

## Intermediate Representation Survival

Zero new markers. The IR that survived P1 — sub-printable integers (`-1`, `1`, `2`) coexisting with ASCII chars on Hamlet's stack — continues to work unchanged. Emphasis passes through as raw `*` (42) and `_` (95) characters and is dispatched in Phase 3 exactly like backtick. Blockquote open/close *bypasses* the marker scheme entirely and pushes raw HTML tag characters onto the same buffer; those chars are sub-dispatched by `@EMIT_CHAR` as regular content. This is a feature: it means the IR does not grow as we add block-level HTML wrappers, provided the wrapper's characters don't collide with existing markers (they don't — all HTML tag chars are ≥ 32).

Marker table unchanged from P1:
- `-1` — bottom sentinel / EOF
- `1` — paragraph-open → `<p>`
- `2` — paragraph-close → `</p>\n\n`
- `96` — literal backtick (also acts as code toggle)

## Block/Inline Handoff

The handoff is exactly the same as P1: Phase 2 pushes a sequence of (markers + chars) onto Hamlet's stack; `@FLIP3` reverses onto Oberon; `@EMIT_LOOP` dispatches on each popped value.

What P2 exercises about the handoff:

- **Inline markers (`*`, `_`) flow through Phase 2 untouched.** Phase 2 treats them as regular non-newline chars in `@NOT_NL` / `@JUST_PUSH`. Phase 3 sees them as dispatch targets. No coordination between phases is needed — they share a value-range convention and that's enough.
- **Block wrappers (`<blockquote>`/`</blockquote>`) are expanded at Phase 2 push time.** Phase 3 knows nothing about blockquotes. This pushes block-level structural responsibility entirely into Phase 2 while keeping Phase 3 a flat char+marker dispatcher. Good separation.
- **State carries across phases via character values.** Mercutio's `1` flag set in Phase 2 `@SEED_BQ_EMIT` survives to Phase 2 `@FINISH` for close-tag dispatch. No on-stack state needed.

## Surprises

- **Inline prose between statements is not legal.** An early draft had English-comment blocks like `                    Peek the first input char to detect '>'.` between a question and the `If so …` conditional. shakespearelang parsed this as `negative_if` and failed with `expecting 'If'`. Only `Scene LABEL: description.` headers and blank lines are accepted between statements. Removed all free-form prose; scene headers carry intent instead.
- **SPL scenes fall through.** Originally `@FINISH` ended with `[Exeunt]` and `@FINISH_BQ` was a separate scene reached by conditional jump. Non-blockquote runs executed `@FINISH` → `[Exeunt]` → then fell through into `@FINISH_BQ`'s `Mercutio: …` speeches with Mercutio off-stage, producing `Mercutio is not on stage!`. Fix: add an explicit `@FINISH_END` tail scene and guard the blockquote path with `If not, … proceed to @FINISH_END`. Plus, for non-BQ, jump *past* `@FINISH_BQ` to `@FINISH_END` via `Let us return to scene @FINISH_END`.
- **Paragraph-close marker emits `</p>\n\n`.** Reusing it for blockquote gave `</p>\n\n</blockquote>` — an unwanted blank line between `</p>` and `</blockquote>`. Fix: skip the `2` marker on the blockquote path and push raw `</p>\n</blockquote>\n` chars instead. The oracle's blockquote output has exactly one newline between `</p>` and `</blockquote>`.
- **State character values persist off-stage.** Mercutio's `1` set during Phase 2 blockquote detection survived all the way through Phase 2 `@FINISH` into Phase 3, where it would have flipped the first `*` into a close instead of an open. Explicit reset in `@FINISH_END` is required. Generalisation: any stage-swap state character that gets set in phase *N* must be either reset at the end of phase *N* or explicitly consumed in a later phase. Treat this as a standing invariant.

Easier than expected:

- **Emphasis mirroring code-toggle.** Copy-pasting the Ophelia/`@TOGGLE_CODE` pattern with Mercutio and new value phrases produced a working `@TOGGLE_EM` first try. The stage-swap-for-state-read idiom generalises cleanly.
- **Blockquote via raw-char push.** Originally I considered adding new IR markers (`3` = blockquote-open, `4` = blockquote-close) plus matching dispatch in Phase 3. Instead, pushing raw HTML tag chars from Phase 2 keeps Phase 3 unchanged — a much smaller change and a reusable trick for any future block wrapper (e.g. `<ul>`/`</ul>`, `<pre>`, headings).

## Items Fed Forward to the Detailed Spec

- **Inline-state character pattern scales.** One character per mutually-exclusive inline state (code, em, strong, …). Each adds one new cast entry, three scenes, and roughly 30 lines of `Speak your mind!` pairs for its open/close HTML tag. Budget accordingly.
- **Block wrappers = Phase 2 raw-char emit, not new IR markers.** For any wrapper whose open/close bytes are all ≥ 32, Phase 2 should push the raw tag characters in forward order and rely on Phase 3's `@EMIT_CHAR` to output them. Only reach for a new marker when a wrapper needs to encode *state* or *variable content* at emit time.
- **State-reset discipline.** Any stage-swap state character set in an earlier phase must be reset before the phase that depends on it starting from zero. The detailed spec should list the reset points explicitly.
- **Markdown.pl emphasis backtracking is a divergence, not a bug to fix.** Matching `<em>foo <strong>bar</em> baz</strong>` requires either backtrack-aware parsing or a CommonMark-style rule rewrite. Both exceed the scope of a stack-machine port. Add to `docs/markdown/divergences.md` rather than designing around it.
- **Free-form comments are banned.** Only `Scene LABEL: description.` headers. Code-gen tooling should refuse to emit anything else between statements.
- **SPL scene fall-through is load-bearing.** Code must be laid out such that reaching the end of a scene always leads to a well-defined next scene (via explicit `Let us …` jumps or by design-time ordering). The detailed spec should note this and prefer explicit jumps at every scene tail.
