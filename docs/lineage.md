# Lineage

This repository is part of a three-step Markdown.pl porting lineage:

- `shakedown` was the SPL attempt
- `snarkdown` was the CURSED attempt
- `quackdown` is the DuckDB SQL implementation

The three projects share the same goal, the same 23 `Markdown.mdtest` fixtures, and the same Huntley-loop methodology. What changed across the lineage was the target runtime and the feasibility lesson each one exposed.

## Why This Repo Exists

This repo is the working home for the next Shakedown implementation effort. It keeps the provenance docs close by so the design and plan can be written with the earlier experiments in view, without inheriting the working clutter of the older repo.

## Local Reference

If you have a sibling `quackdown` checkout locally, it may contain the original Shakedown research docs and historical notes. This repo is self-contained, but that sibling checkout can be used as a convenience reference for the original feasibility work.
